## Overview

The Interaction Logs API allows you to record notes about your follow-up interactions with contacts. When you create an interaction log for a follow-up task, the task is automatically marked as complete. All interaction logs are tied to a specific follow-up task, contact, and user.

## Create Interaction Log

<CodeGroup>
```bash POST /follow_up_tasks/:follow_up_task_id/interaction_logs
curl -X POST "https://api.trellixe.com/follow_up_tasks/23/interaction_logs" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "interaction_log": {
      "note": "Called to follow up on product interest. They want to schedule a demo next week."
    }
  }'
```
</CodeGroup>

Create a new interaction log for a follow-up task. This automatically marks the task as complete.

### Path Parameters

<ParamField path="follow_up_task_id" type="integer" required>
  The ID of the follow-up task to log an interaction for
</ParamField>

### Request Body

<ParamField body="interaction_log" type="object" required>
  <ParamField body="note" type="string" required>
    Notes about the interaction with the contact (required)
  </ParamField>
</ParamField>

### Response

<CodeGroup>
```json 201 Created
{
  "id": 42,
  "follow_up_task_id": 23,
  "contact_id": 1,
  "user_id": 5,
  "note": "Called to follow up on product interest. They want to schedule a demo next week.",
  "created_at": "2026-03-16T10:30:00Z",
  "updated_at": "2026-03-16T10:30:00Z",
  "message": "Follow-up successfully logged!"
}
```

```json 422 Unprocessable Entity
{
  "errors": {
    "note": ["can't be blank"]
  }
}
```
</CodeGroup>

### Automatic Task Completion

When you create an interaction log, two things happen in a single database transaction:

1. The interaction log is saved
2. The associated follow-up task's `completed_at` is set to the current timestamp

If either operation fails, both are rolled back to maintain data consistency.

## Interaction Log Model

<ResponseField name="id" type="integer">

## Security & Access Control

Interaction logs have strict security controls:

### User Scoping

You can only create interaction logs for follow-up tasks assigned to you:

```ruby
@follow_up_task = current_user.follow_up_tasks.find(params[:follow_up_task_id])
```

Attempting to create a log for someone else's task will result in a 404 error.

### Automatic Attribute Setting

The following attributes are automatically set and cannot be overridden:

- `contact_id` - Set from the follow-up task's invitation
- `user_id` - Set to the current user
- `follow_up_task_id` - Set from the URL path parameter

## Workflow Example

### Complete Follow-Up Flow

1. **Event Attendance**
   - Contact attends "Product Launch Party" on March 15, 2026
   - User marks invitation as "attended"

2. **Automatic Task Creation**
   - Follow-up task created, due March 16, 2026 at 9:00 AM
   - Reminder job scheduled

3. **Reminder Notification**
   - User receives notification on March 16 at 9:00 AM
   - "Time to follow up with John Doe from Product Launch Party"

4. **Follow-Up & Logging**
   - User calls John to discuss product interest
   - User creates interaction log via POST request

5. **Automatic Completion**
   - Interaction log saved
   - Follow-up task marked as complete
   - Task removed from pending list

### Example Request Flow

<CodeGroup>
```bash 1. Get Task Details
curl -X GET "https://api.trellixe.com/follow_up_tasks" \
  -H "Authorization: Bearer YOUR_API_TOKEN"

# Response includes task ID 23 for John Doe
```

```bash 2. View Contact Context
curl -X GET "https://api.trellixe.com/follow_up_tasks/23/interaction_logs/new" \
  -H "Authorization: Bearer YOUR_API_TOKEN"

# Response includes contact details and event info
```

```bash 3. Log Interaction
curl -X POST "https://api.trellixe.com/follow_up_tasks/23/interaction_logs" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "interaction_log": {
      "note": "Discussed partnership opportunities. Sending proposal next week."
    }
  }'

# Task 23 is now marked as complete
```

```bash 4. Verify Completion
curl -X GET "https://api.trellixe.com/follow_up_tasks" \
  -H "Authorization: Bearer YOUR_API_TOKEN"

# Task 23 no longer appears in pending tasks
```
</CodeGroup>

## Transaction Safety

Creating an interaction log uses an `ActiveRecord::Base.transaction` to ensure data consistency:

```ruby
ActiveRecord::Base.transaction do
  @interaction_log.save!
  @follow_up_task.update!(completed_at: Time.current)
end
```

If either operation fails:
- Both operations are rolled back
- No data is saved
- An error response is returned with validation details

This ensures you never have an interaction log without a completed task, or a completed task without a log.
